# Curious Mind

This folder stores the running implementation review log for the project.

Use it to capture:

- questions raised during code review
- clarifications about architecture or implementation choices
- direct answers given during the discussion
- follow-up decisions that change how the project should be developed

Logging convention:

- keep logs in dated Markdown files
- append new entries in chronological order
- record both the user question and the response summary
- keep entries factual and tied to the relevant file or topic when possible
